# 冷启动验证提示词

> 把这个提示词和 SPEC.md、PLAN.md 一起交给一个**不同的** agent（不要用本 agent），在一个全新 session 中执行。

---

你是一个冷启动验证 agent。你的任务是**仅凭 SPEC.md 和 PLAN.md 两个文件**，尝试实现 PLAN 中的 1-2 个 task。

## 约束

1. **无额外上下文**：你没有参与过 brainstorming 讨论，不知道任何对话历史。所有信息只能从 SPEC.md 和 PLAN.md 中获取。
2. **自主推进**：选择 PLAN 中的 1-2 个 task 自主推进，完成 TDD 红绿循环。
3. **遇到不确定处暂停**：如果 spec 或 plan 中写得不清楚，导致你无法判断"该怎么做"，**暂停并提问**，而非凭猜测继续。
4. **记录暂停点**：记下你在哪里暂停、为什么暂停、你期望 spec/plan 中应该有什么信息。

## 工作方式

1. 先阅读 SPEC.md 和 PLAN.md
2. 选择你认为最独立的 task（如 Task 2 数据模型、Task 3 配置加载）
3. 按 TDD 流程执行：先写测试 → 跑测试确认失败 → 写实现 → 跑测试确认通过
4. 完成后汇报：完成了哪些 task、遇到了哪些问题、spec 中哪些地方写得不清楚、你的修改建议
